# Fixing Action Text Format Specification

This document defines the exact text format for each action type in the "Fixing Action" column.
These formats are designed to be parsed by downstream PCF builders using regex extraction.

## Action Formats

### 1. TRIM
**Triggered by:** Overlap with matching bores

**Format:**
```
TRIM: Reduce by <N>mm
  Endpoint <1|2>: Move to intersection
  New coord: (<x>, <y>, <z>)
  Overlap with <TYPE> resolved
```

**Regex Patterns:**
- `/Endpoint\s+(\d+):/` → Captures endpoint index (1 or 2)
- `/New coord:\s*\(([^)]+)\)/` → Captures coordinates as "x, y, z"

**Example:**
```
TRIM: Reduce by 25.50mm
  Endpoint 2: Move to intersection
  New coord: (1500.00, 0.00, 0.00)
  Overlap with PIPE resolved
```

---

### 2. SNAP
**Triggered by:** Gap ≤ 6mm (broken connection)

**Format:**
```
SNAP: Merge endpoints to midpoint
  <TYPE> EP2: Move <N>mm → (<x>, <y>, <z>)
  <TYPE> EP1: Move <N>mm → (<x>, <y>, <z>)
```

**Regex Patterns:**
- `/EP(\d):\s*Move.*?→\s*\(([^)]+)\)/g` → Captures endpoint (1 or 2) and target coordinates

**Example:**
```
SNAP: Merge endpoints to midpoint
  PIPE EP2: Move 1.75mm → (1001.75, 0.00, 0.00)
  PIPE EP1: Move 1.75mm → (1001.75, 0.00, 0.00)
```

---

### 3. EXTEND
**Triggered by:** Model error with gap ≤ 6mm

**Format:**
```
SNAP: Close <N>mm gap (below tolerance)
  <TYPE> EP2: Extend <N/2>mm
  <TYPE> EP1: Extend <N/2>mm
```

**Regex Patterns:**
- `/EP(\d):\s*Extend ([0-9.]+)mm/g` → Captures endpoint and extension distance

**Example:**
```
SNAP: Close 4.00mm gap (below tolerance)
  PIPE EP2: Extend 2.00mm
  PIPE EP1: Extend 2.00mm
```

**Note:** Direction determined from EP1→EP2 unit vector, extend by N/2 along that vector.

---

### 4. INSERT PIPE
**Triggered by:** Gap > 6mm (broken connection)

**Format:**
```
INSERT PIPE: Fill <N>mm gap
  New component: PIPE
  EP1: (<x>, <y>, <z>)
  EP2: (<x>, <y>, <z>)
  Length: <N>mm, Bore: <B>mm
```

**Regex Patterns:**
- `/EP1:\s*\(([^)]+)\)/` → Captures EP1 coordinates
- `/EP2:\s*\(([^)]+)\)/` → Captures EP2 coordinates
- `/Bore:\s*([0-9.]+)mm/` → Captures bore diameter

**Example:**
```
INSERT PIPE: Fill 550.00mm gap
  New component: PIPE
  EP1: (1000.00, 0.00, 0.00)
  EP2: (1550.00, 0.00, 0.00)
  Length: 550.00mm, Bore: 273.00mm
```

**Important:** De-duplicate - only insert ONE new row when two consecutive rows share this text.

---

### 5. FILL GAP
**Triggered by:** Gap > 2×bore (model error)

**Format:**
```
FILL GAP: Insert connector for <N>mm gap
  Gap exceeds 2×bore threshold
  From: (<x>, <y>, <z>)
  To: (<x>, <y>, <z>)
```

**Regex Patterns:**
- `/From:\s*\(([^)]+)\)/` → Captures EP1 coordinates
- `/To:\s*\(([^)]+)\)/` → Captures EP2 coordinates
- `/for\s*([0-9.]+)mm gap/` → Captures gap distance

**Example:**
```
FILL GAP: Insert connector for 5000.00mm gap
  Gap exceeds 2×bore threshold
  From: (1000.00, 0.00, 0.00)
  To: (6000.00, 0.00, 0.00)
```

**Note:** Bore fallback from nearest component.

---

### 6. DELETE ROW
**Triggered by:** Zero-length orphan component with no connections

**Format:**
```
DELETE ROW: Component has zero length and no connections
```

**Detection:**
- `startsWith('DELETE ROW')`

**Action:**
- Remove component from cloned array

---

### 7. REVIEW REQUIRED
**Triggered by:** Overlap with different bores (manual review needed)

**Format:**
```
REVIEW REQUIRED: <N>mm overlap detected
  <TYPE> (bore <B>mm)
  <TYPE> (bore <B>mm)
  Different bores - manual review needed
```

**Detection:**
- `startsWith('REVIEW REQUIRED')`

**Action:**
- SKIP - no modification, requires manual intervention

**Example:**
```
REVIEW REQUIRED: 15.20mm overlap detected
  PIPE (bore 273.00mm)
  PIPE (bore 219.10mm)
  Different bores - manual review needed
```

---

## Implementation Notes

1. **Coordinate Format:** All coordinates use 2 decimal places: `(x.xx, y.yy, z.zz)`
2. **Distance Format:** All distances use 2 decimal places: `N.NNmm`
3. **Bore Format:** All bore diameters use 2 decimal places: `B.BBmm`
4. **Component Types:** Default to `PIPE` if unknown
5. **Endpoint Indexing:** 1-based in text (EP1, EP2), convert to 0-based for array access
6. **Whitespace:** Two spaces for indentation on continuation lines
7. **Newlines:** Use `\n` for line breaks within action text

---

## Validation

All action descriptions are generated by `ActionDescriptor.js` and must:
- Match the exact format specified above
- Be parseable by the regex patterns listed
- Provide all information needed for downstream PCF regeneration
- Support de-duplication for INSERT PIPE actions
